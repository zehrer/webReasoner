# Deployment

Web Reasoner is deployed to GitHub Pages from the `gh-pages` branch. The
published files are generated by the GitHub Actions workflow in
`.github/workflows/deploy.yml`.

## Workflow

The workflow is named `Deploy to gh-pages`.

It runs when:

- a commit is pushed to `main`
- the workflow is started manually from the GitHub Actions tab

The workflow does the following:

1. Checks out the repository.
2. Installs the current Node.js LTS release.
3. Installs dependencies with `npm ci`.
4. Builds the static site with `npm run build`.
5. Creates `dist/.nojekyll` so GitHub Pages serves Vite assets unchanged.
6. Publishes the `dist` folder to the `gh-pages` branch.

The deploy step uses `JamesIves/github-pages-deploy-action@v4` with
`clean: true`, so each deployment replaces the previous branch contents with
the latest build output.

## GitHub Pages Settings

In the repository settings, configure GitHub Pages like this:

- Source: `Deploy from a branch`
- Branch: `gh-pages`
- Folder: `/root`

After the first successful workflow run, GitHub creates or updates the
`gh-pages` branch automatically.

## Vite Base Path

`vite.config.ts` sets:

```ts
base: "/webReasoner/",
```

This is required because the project is served from a GitHub Pages project URL,
not from the domain root. Without this setting, built assets would be referenced
from `/assets/...` and fail to load on:

```text
https://zehrer.github.io/webReasoner/
```

If the repository name or Pages path changes, update the Vite `base` value to
match the new path.

## Local Verification

Before pushing deployment changes, run:

```bash
npm run build
```

After pushing to `main`, check the workflow status:

```bash
gh run list --repo zehrer/webReasoner --limit 3
```
